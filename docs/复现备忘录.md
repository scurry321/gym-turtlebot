# gym-turtlebot 复现备忘录

> 记录从零开始复现本项目所需的完整步骤。
> 目标平台：Ubuntu 24.04 + ROS 2 Jazzy + Gazebo Harmonic + Python 3.12
>
> **当前系统为 Ubuntu 22？** → 推荐使用 [Docker + VS Code Dev Container 方案](#方案-b-docker--vs-code-dev-container推荐) 跳过系统级安装，直接进入开发。

## 方案 B：Docker + VS Code Dev Container（推荐）

> 项目已内置 `.devcontainer/` 配置，基于 `althack/ros2:jazzy-dev`（Ubuntu 24 + ROS 2 Jazzy），
> 在 **Ubuntu 22 / Windows WSL2 / macOS** 上均可使用，无需手动安装 ROS 2。

### B.1 安装前提

- [ ] 安装 [Docker Engine](https://docs.docker.com/engine/install/ubuntu/)

```bash
# 快速安装 Docker（Ubuntu 22）
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
    | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" \
    | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin -y

# 把当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER
newgrp docker
```

- [ ] 安装 [VS Code](https://code.visualstudio.com/) + 插件 [Dev Containers](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers)

```bash
# 安装 Dev Containers 插件
code --install-extension ms-vscode-remote.remote-containers
```

### B.2 开启 X11 GUI 转发（在 Ubuntu 22 宿主机上）

容器内需要显示 Gazebo / RViz GUI，执行一次：

```bash
xhost +local:docker
```

> 若使用 Wayland 桌面，还需确认 `WAYLAND_DISPLAY` 已设置；`devcontainer.json` 已自动透传该变量。

### B.3 在 VS Code 中打开 Dev Container

1. 用 VS Code 打开项目目录 `gym-turtlebot/`
2. 按 `Ctrl+Shift+P` → 输入 `Dev Containers: Reopen in Container` → 回车
3. VS Code 将自动 **构建镜像**（首次约 5–10 分钟）并进入容器环境

镜像内容（由 `.devcontainer/Dockerfile` 定义）：
| 内容 | 版本 |
|------|------|
| 基础镜像 | `althack/ros2:jazzy-dev`（Ubuntu 24.04） |
| ROS 2 | Jazzy（含 colcon、rosdep） |
| Gazebo | Harmonic（`ros-jazzy-ros-gz`） |
| 环境变量 | `PIP_BREAK_SYSTEM_PACKAGES=1`，自动 source 工作空间 |

### B.4 容器内完成余下步骤

进入容器后，**直接跳到 [阶段二](#阶段二克隆项目并安装-ros-依赖)**，注意：
- 阶段一（系统安装）**完全跳过**
- 项目目录已自动挂载到容器内的工作空间目录
- `ROS_DISTRO=jazzy` 已预设，无需手动 source `/opt/ros/jazzy/setup.bash`

### B.5 常见 Docker 问题

| 问题 | 解决方案 |
|------|---------|
| Gazebo 黑屏 / 无法打开窗口 | 宿主机执行 `xhost +local:docker`，并确认 `DISPLAY` 变量传入容器 |
| 容器内 `gz sim` 崩溃 | `devcontainer.json` 已设置 `LIBGL_ALWAYS_SOFTWARE=1`，软件渲染，无需 GPU |
| 重建镜像（依赖变更后）| `Ctrl+Shift+P` → `Dev Containers: Rebuild Container` |
| 容器内无网络 | `devcontainer.json` 已配置 `--network=host`，与宿主机共享网络 |

## 整体架构概览

```
gym-turtlebot/
├── src/
│   ├── tb4_drl_navigation/   # Python 包：Gymnasium 环境 + SAC 训练示例
│   └── turtlebot4/
│       ├── tb4_description/  # CMake 包：TurtleBot4 URDF/mesh/xacro
│       └── tb4_gz_sim/       # CMake 包：Gazebo 仿真 launch 文件 + 世界模型
├── build.sh                  # colcon 构建脚本
├── setup.sh                  # rosdep 依赖安装脚本
└── pyproject.toml            # Python 依赖声明
```

**数据流**：训练脚本 → Gymnasium 环境 (`turtlebot4.py`) → ROS 2 话题/服务 → Gazebo 仿真

## 阶段一：系统环境准备（本地安装，使用 Docker 可跳过此阶段）

### 1.1 安装 Ubuntu 24.04

- [ ] 安装 Ubuntu 24.04 LTS（Desktop 或 Server 均可）
- [ ] 更新系统包

```bash
sudo apt update && sudo apt upgrade -y
```

### 1.2 安装 ROS 2 Jazzy

官方文档：<https://docs.ros.org/en/jazzy/Installation.html>

```bash
# 设置 locale
sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# 添加 ROS 2 apt 源
sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
    -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
    http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
    | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# 安装 ROS 2 Jazzy 完整桌面版
sudo apt update
sudo apt install ros-jazzy-desktop -y

# 安装构建工具
sudo apt install python3-colcon-common-extensions python3-rosdep -y
```

- [ ] 验证安装

```bash
source /opt/ros/jazzy/setup.bash
ros2 --version  # 应输出 ros2 jazzy
```

### 1.3 安装 Gazebo Harmonic

官方文档：<https://gazebosim.org/docs/harmonic/ros_installation/>

```bash
sudo apt-get install ros-jazzy-ros-gz -y
```

- [ ] 验证安装

```bash
gz sim --version  # 应输出 Gazebo Sim, version 8.x.x (Harmonic)
```

### 1.4 安装 rosdep 并初始化

```bash
sudo rosdep init
rosdep update --rosdistro=jazzy
```

## 阶段二：克隆项目并安装 ROS 依赖

### 2.1 克隆仓库

```bash
cd ~
git clone https://github.com/anurye/gym-turtlebot.git
cd gym-turtlebot
```

- [ ] 确认目录结构包含 `src/tb4_drl_navigation` 和 `src/turtlebot4`

### 2.2 配置 ROS 环境变量（每次新终端都需要）

```bash
source /opt/ros/jazzy/setup.bash
export ROS_DISTRO=jazzy
```

> **提示**：可将以下内容加入 `~/.bashrc` 避免每次手动 source：
> ```bash
> echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
> echo "export ROS_DISTRO=jazzy" >> ~/.bashrc
> ```

### 2.3 安装 ROS 包依赖（运行 setup.sh）

`setup.sh` 等价于：

```bash
sudo apt-get update
rosdep update --rosdistro=$ROS_DISTRO
rosdep install --from-paths src --ignore-src -y --rosdistro=$ROS_DISTRO
```

项目所需的关键 ROS 包依赖：

| 包名 | 来源 |
|------|------|
| `ros-jazzy-slam-toolbox` | apt |
| `ros-jazzy-robot-localization` | apt |
| `ros-jazzy-ros-gz-bridge` | apt（ros-jazzy-ros-gz 已含） |
| `ros-jazzy-ros-gz-interfaces` | apt |
| `ros-jazzy-ros-gz-sim` | apt |
| `ros-jazzy-robot-state-publisher` | apt |
| `ros-jazzy-xacro` | apt |

```bash
./setup.sh
```

- [ ] 确认 rosdep 无报错退出

## 阶段四：构建 ROS 工作空间

```bash
cd ~/gym-turtlebot
source /opt/ros/jazzy/setup.bash
./build.sh
```

`build.sh` 实际执行的是：

```bash
colcon build \
    --merge-install \
    --symlink-install \
    --cmake-args "-DCMAKE_BUILD_TYPE=RelWithDebInfo" \
    -Wall -Wextra -Wpedantic
```

- [ ] 构建无报错（警告可忽略）
- [ ] 确认生成 `install/` 和 `build/` 目录

构建完成后 source 工作空间：

```bash
source install/local_setup.bash
```

> **提示**：可加入 `~/.bashrc`：
> ```bash
> echo "source ~/gym-turtlebot/install/local_setup.bash" >> ~/.bashrc
> ```

## 常见问题排查

### Q1: `rosdep install` 失败，找不到某些包

```bash
# 检查 rosdep 是否已初始化
cat /etc/ros/rosdep/sources.list.d/20-default.list
# 重新更新 rosdep
rosdep update --rosdistro=jazzy
```

### Q2: colcon build 失败，提示找不到 slam_toolbox

```bash
sudo apt install ros-jazzy-slam-toolbox -y
```

### Q3: Gazebo 启动黑屏 / 渲染异常

```bash
# 尝试软件渲染模式（无 GPU 时）
export LIBGL_ALWAYS_SOFTWARE=1
ros2 launch tb4_gz_sim simulation.launch.py
```

### Q4: 机器人重置位置不对

项目使用两个服务进行复位（参见 README 说明）：
1. `robot_localization` 的 `SetPose` 服务 → 重置里程计
2. `ros_gz_interfaces` 的 `SetEntityPose` 服务 → 重置 Gazebo 中实体位置

如果复位异常，检查这两个服务是否正常运行：

```bash
ros2 service list | grep -E "set_pose|SetEntityPose"
```

### Q5: Python import 报错 `No module named 'tb4_drl_navigation'`

```bash
source install/local_setup.bash
# 或者确认 PYTHONPATH 包含工作空间
echo $PYTHONPATH
```

### Q6: `gymnasium` 版本冲突

```bash
pip install "gymnasium>=1.1.1" --upgrade
```
