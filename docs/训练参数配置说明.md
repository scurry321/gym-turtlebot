# 训练参数配置说明

训练脚本位于 [src/tb4_drl_navigation/tb4_drl_navigation/examples/sac.py](../src/tb4_drl_navigation/tb4_drl_navigation/examples/sac.py)，所有参数通过三个 dataclass 集中管理：`EnvConfig`、`SACConfig`、`ExperimentConfig`。

---

## 快速上手

### 全新训练
```bash
source install/setup.bash
ros2 run tb4_drl_navigation example train
```

### 从检查点续训
```bash
source install/setup.bash
ros2 run tb4_drl_navigation example train \
  --resume experiments/sac_navigation/checkpoints/sac_model_40000_steps.zip
```

### 推理评估
```bash
source install/setup.bash
ros2 run tb4_drl_navigation example eval \
  experiments/sac_navigation/final_model.zip \
  --episodes 20
```

## 参数说明

### 1. 环境参数 (`EnvConfig`)

对应 `config.yaml` 中的 `env:` 节点，控制仿真环境行为。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `env_id` | `Turtlebot4Env-v0` | Gymnasium 环境 ID，不建议修改 |
| `world_name` | `static_world` | Gazebo 世界名称，需与 SDF 文件名一致 |
| `robot_name` | `turtlebot4` | 仿真中机器人模型名称 |
| `obstacle_prefix` | `obstacle` | 障碍物模型名称前缀（如 `obstacle_1`、`obstacle_2`…） |
| `robot_radius` | `0.3` | 机器人碰撞半径（米），用于碰撞检测 |
| `obstacle_clearance` | `2.0` | 目标点与障碍物的最小距离（米），防止目标点生成在障碍物内 |
| `min_separation` | `1.5` | 机器人起点与目标点的最小距离（米） |
| `goal_sampling_bias` | `close` | 目标点采样策略：`close`（偏近距离）或 `uniform`（均匀采样） |
| `num_bins` | `20` | LiDAR 激光扫描离散化的区间数，影响观测空间维度 |
| `goal_threshold` | `0.35` | 判定到达目标的距离阈值（米） |
| `collision_threshold` | `0.4` | 判定碰撞的距离阈值（米） |
| `time_delta` | `0.025` | 每个仿真步的时间间隔（秒），值越小仿真越精细但速度越慢 |
| `shuffle_on_reset` | `true` | 每次 reset 时是否随机打乱障碍物位置 |
| `map_path` | `null` | 自定义地图路径（可选） |
| `yaml_path` | `null` | 自定义地图配置 YAML 路径（可选） |
| `sim_launch_name` | `null` | 自定义仿真 launch 文件路径（可选） |

**调参建议：**
- 提高训练难度：减小 `goal_threshold`、增大 `obstacle_clearance`
- 加快仿真速度：增大 `time_delta`（0.05~0.1），但物理精度会下降
- 固定场景（方便调试）：设置 `shuffle_on_reset: false`

---

### 2. SAC 算法参数 (`SACConfig`)

对应 `config.yaml` 中的 `sac:` 节点，控制 Soft Actor-Critic 算法超参数。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `policy_type` | `MlpPolicy` | 策略网络类型，使用全连接网络 |
| `buffer_size` | `1_000_000` | Replay Buffer 容量（transition 数），越大越占内存 |
| `batch_size` | `1024` | 每次梯度更新使用的样本数 |
| `gamma` | `0.99` | 折扣因子，控制对未来奖励的重视程度（0~1） |
| `tau` | `0.005` | 目标网络软更新系数，越小目标网络越稳定 |
| `target_update_interval` | `1` | 目标网络更新频率（每 N 步更新一次） |
| `learning_rate` | `3e-4` | Adam 优化器学习率 |
| `train_freq` | `(1, "step")` | 每收集多少步/episode 执行一次训练 |
| `gradient_steps` | `4` | 每次训练执行的梯度更新次数 |

**网络结构 (`policy_kwargs`)：**

```yaml
policy_kwargs:
  net_arch:
    pi: [512, 512, 256]   # Actor 网络各层宽度
    qf: [1024, 1024, 512] # Critic 网络各层宽度
  activation_fn: ReLU
  optimizer_class: Adam
```

**调参建议：**
- 样本效率不足：增大 `gradient_steps`（4→8）或减小 `train_freq`
- 训练不稳定：降低 `learning_rate`（3e-4→1e-4），增大 `tau`（0.005→0.01）
- 内存不足：减小 `buffer_size`（1_000_000→500_000）
- 网络容量不够（任务复杂）：增大 `net_arch` 各层宽度

---

### 3. 实验参数 (`ExperimentConfig`)

对应 `config.yaml` 顶层字段，控制训练流程。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `total_timesteps` | `500_000` | 总训练步数 |
| `save_freq` | `5000` | 每隔多少步保存一次检查点（同时触发评估） |
| `eval_freq` | `5000` | 每隔多少步评估一次（当前与 `save_freq` 相同） |
| `n_eval_episodes` | `5` | 每次评估运行的 episode 数 |
| `seed` | `42` | 随机种子，设为 `null` 则不固定随机性 |
| `use_deterministic_cudnn` | `false` | 是否使用确定性 cuDNN（影响 GPU 训练的可复现性） |
| `log_dir` | `experiments` | 实验输出根目录 |
| `experiment_name` | `sac_navigation` | 实验名称，输出保存到 `{log_dir}/{experiment_name}/` |

## 输出目录结构

```
experiments/sac_navigation/
├── checkpoints/                     # 定期检查点
│   ├── sac_model_5000_steps.zip
│   ├── sac_model_5000_steps_replay_buffer.pkl
│   ├── sac_model_10000_steps.zip
│   └── ...
├── best_model/                      # 评估指标最优的模型
│   └── best_model.zip
├── eval_logs/                       # 评估日志（numpy 格式）
├── logs/                            # TensorBoard 日志
│   └── sac_training_1/
├── monitor.csv                      # 每个 episode 的奖励/步数记录
├── final_model.zip                  # 训练结束/中断时保存的最终模型
├── final_model_replay_buffer.pkl    # 对应的 Replay Buffer
└── config.yaml                      # 本次训练使用的完整参数快照
```

## 修改参数的方法

直接修改 [sac.py](../src/tb4_drl_navigation/tb4_drl_navigation/examples/sac.py) 中对应 dataclass 的默认值，然后重新 build：

```bash
./build.sh
```

**示例：将总步数改为 1,000,000 步**

```python
# ExperimentConfig
total_timesteps: int = 1_000_000
```

**示例：加快仿真并减少 LiDAR 分辨率**

```python
# EnvConfig
time_delta: float = 0.1   # 加快仿真
num_bins: int = 10         # 减少观测维度
```

## 查看训练曲线

```bash
source .venv/bin/activate
tensorboard --logdir experiments/sac_navigation/logs
```

然后在浏览器打开 `http://localhost:6006`。

监控 `monitor.csv` 中的 `r` 列（episode 奖励）也可快速判断训练进展：奖励从约 -100 开始，成功到达目标的回合奖励约为 +100。
